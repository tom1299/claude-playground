# Coding still for shell scripts
* Avoid complicated regexes, use simple string manipulation where possible
* Avoid complicated if statements
* Avoid one line if statements. Use multiple lines for clarity
* Use functions to break up code into smaller, reusable pieces
* Keep cognitive complexity by applying the following rules:
  * Avoid nested loops and conditionals
  * Limit the number of branches in if statements
  * Avoid deep nesting of functions
  * Use clear and descriptive variable names
  * Avoid long functions, break them into smaller ones
  * Use comments to explain complex logic, but avoid over-commenting
